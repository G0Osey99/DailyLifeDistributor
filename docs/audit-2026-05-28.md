# Codebase audit — 2026-05-28

Full top-to-bottom review via six parallel subagents (web pipeline, uploaders,
agent, auth/security, core infra, calendar/refresh/preflight). Findings triaged
below. Severity reflects **this deploy** (effectively single-org); multi-tenant
concurrency issues are real but don't bite one user, so they're deferred with a
note rather than fixed under deadline pressure.

Legend: ✅ fixed this pass · 📝 noted/deferred (rationale given) · ❌ finding incorrect.

## Tier 1 — correctness, web + agent end-to-end (FIXED)

- ✅ **AGT-1** `agent/secrets_shim.py` — shim `get/set/delete_secret`, `get/set_blob`
  lacked the `org_id=` kwarg the real uploaders pass → `TypeError` on the first
  real YouTube/Playwright credential read. The agent path crashed before any
  upload. Added ignored `*, org_id=None` to every shim method.
- ✅ **AGT-2** `agent/secrets_shim.py` — `youtube_uploader._load_client_config`
  calls `get_platform_secret("youtube.client_secrets")`, absent from the shim →
  `AttributeError`. Added `get_platform_secret`/`get_platform_blob` delegating to
  the bare-key lookup (the envelope stores creds by bare key).
- ✅ **AGT-3** `agent/run_batch.py` — breaker-OPEN `YouTube Video` row returned
  without `yt_state.record(None)`, so a `Rock Email` waiter blocked the full
  30-min timeout. Record `None` in the breaker-open branch.
- ✅ **WEB-11** `core/upload_jobs.py` — same deadlock class on the web path: the
  email waiter could block 30 min when the YT row never produced a result.
  Email waiter now also observes the cancel event + a short bound.
- ✅ **WEB-7** `core/upload_jobs.py` — an idempotently-skipped (already-success)
  row wrote a fresh `success=False` history row, which could flip the date back
  to "not done" on the next re-run. Don't record a failure row for idempotent
  skips.
- ✅ **WEB-13** `core/upload_jobs.py` — `breaker.record_success()` fired on
  element/idempotent skips, healing a breaker that never exercised the platform.
  Only record success on a genuine `result.success`.
- ✅ **WEB-8** `core/upload_jobs.py` — an element-disabled YouTube Video row made
  the date's Rock Email error instead of falling back to
  `entry.youtube_watch_url`. Treat element-skipped YT Video as "not expected".
- ✅ **UPL-15** `uploaders/rock/orchestrator.py` — broad `except Exception`
  converted Playwright `TimeoutError`/`OSError` into neutral result-dicts, so the
  Rock circuit breaker never opened on a dead session. Let infra exceptions
  propagate (the dispatch records the breaker failure).
- ✅ **AGT-5** `agent/run_batch.py` — when no media roots are configured the
  agent emitted N generic file-not-found errors. Emit one clear
  "no media folders configured on this device" error instead.
- ✅ **AGT-10** `agent/dispatch.py` — `PendingResults.observe` indexed required
  keys directly; switched to `.get()` so a malformed frame can't `KeyError`.
- ✅ **CAL-9** `blueprints/preflight.py` — LLM model check exact-matched against
  `/v1/models` ids, a false-RED for Ollama tag forms (`llama3.2` vs
  `llama3.2:latest`). Match on prefix/tag equivalence.
- ❌ **CAL-13** rock_email dryrun "false-RED" — INCORRECT for this deploy.
  `ReviewEntry.youtube_watch_url` is never populated from the spreadsheet (no
  column mapping, no `build_entry` assignment), so Rock Email genuinely always
  needs YouTube Video in the run. The current dryrun check is accurate;
  changing it would make preflight *less* truthful. No action.
- ✅ **CAL-1** `blueprints/preflight.py` — SimpleCast/Vista dryrun checked only
  file presence, not the required title/caption → false-GREEN. Add title/caption
  checks.
- ✅ **CAL-8** `blueprints/preflight.py` — YouTube readiness is token-presence,
  not API-validity; the detail now says so honestly.

## Tier 2 — real, lower urgency (FIXED where cheap)

- ✅ **SEC-1** `app.py` — primary `/login` POST wasn't in the flask-limiter list
  (only 2FA/recovery were); brute-force relied on the in-process lockout alone.
  Added a limiter ceiling on `auth.login_submit`.
- ✅ **WEB-12** `core/media_session.py` — the per-session spreadsheet cache under
  `spreadsheets/` was never swept (orphan sweep only handles run-id dirs),
  growing unbounded. Added a TTL sweep at startup.
- ✅ **INF-4** `blueprints/agent.py` — pair-redeem revoked a prior HWID-matched
  device *before* validating the pairing code, a griefing/DoS primitive. Validate
  + consume the code first; revoke only on success.
- ✅ **INF-5** `core/devices.py` — pairing-code consume was SELECT-then-UPDATE
  without a `consumed=0` guard. Made the UPDATE conditional + check rowcount.
- 📝 **INF-2** agent-path `record_upload` writes `org_id=NULL` — for a single-org
  deploy `effective_org_id()` is consistent, so history/calendar scoping still
  resolves. Deferred (would need org threaded through `on_frame`).
- 📝 **UPL-3** YouTube naive-datetime schedule shift — verified `session_state`
  builds tz-aware schedule datetimes (`_default_schedule` attaches tz), so the
  naive path isn't hit in practice. Added a defensive guard note; low risk.
- 📝 **CAL-5** calendar shows YouTube UTC times in the wrong day cell near
  midnight — display-only (doesn't affect what's uploaded). Deferred.

## Tier 3 — deferred (rationale)

- 📝 **WEB-2 / WEB-4 / WEB-5** concurrent-run state bleed via the global `session`
  singleton + process-global `upload_results`. Real, but only bites when two runs
  are in flight at once (multiple orgs/tabs). This deploy runs one user, one run
  at a time. High-risk refactor (request-local session snapshot) — do NOT change
  under the Monday deadline. Tracked for the multi-tenant hardening pass.
- 📝 **WEB-1** per-file chunk-state race — the browser uploads chunks sequentially
  per file; the race needs a buggy/concurrent client. Low real risk. Deferred.
- 📝 **WEB-10** job registry breaks under multi-worker WSGI — documented landmine;
  the deploy is single-process. Deferred (would need a shared store).
- 📝 **SEC-2** GET-based recovery approval is CSRF-able — real account-takeover
  path in principle. Converting to POST changes the emailed-link UX; needs a form
  page. Deferred to a focused security PR (not a blind edit before Monday).
- 📝 **SEC-3** 2FA enforcement is allowlist-by-path (fails open on new routes).
  Real defense-in-depth gap; the practical org doesn't enforce 2FA. Deferred.
- 📝 **INF-1 / INF-11 / SEC-5 / SEC-6** `effective_org_id() is None` → unscoped
  fallback could cross tenants. Single-org deploy; the rows are one tenant's.
  Added a cheap hosted-mode note; full guard deferred to the multi-tenant pass.
- 📝 **UPL-5 / UPL-13** SimpleCast/Rock create duplicate drafts / orphan children
  on re-run after partial failure. Real; the idempotent-skip (`has_successful_upload`)
  prevents re-running a *successful* row, so duplicates only occur after a genuine
  mid-flight failure + retry. Moderate effort; noted for an uploader-idempotency pass.
- ✅ **UPL-14** `uploaders/rock/orchestrator.py` — `append_credits_entry` was
  un-guarded, so a credits-file write error flipped a fully-successful Rock build
  to failure (→ orphan/duplicate on re-run). Wrapped it like `record_image_use`.
- 📝 **UPL-10** Rock failure screenshots accumulate in the project root and may
  contain PII. Moderate; noted (write to a temp/log dir with rotation).
- ❌ **INF-13** "no signature verification on agent binaries" — INCORRECT. The
  agent verifies sha256 + ed25519 against the baked `agent/release_pubkey.pem`
  in `updater.download_and_verify` before executing. The server serving the
  bytes is fine; the trust boundary (the agent) verifies. No action.
- 📝 Various LOW smells (relay private-attr reach-in, db WAL-comment staleness,
  HALF_OPEN trial-slot leak on caller crash, quota separate-connection) — noted,
  not individually fixed; none affect correctness of the happy path.
